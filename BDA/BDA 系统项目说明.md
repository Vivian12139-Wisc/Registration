## 1. 系统概览

BDA 考试管理系统处理舞蹈考试从在线报名到成绩发布的完整周期，主要分为以下五个阶段：

| 阶段 | 内容 |
|---|---|
| **在线报名** | 申请人填写资料、上载文件、缴费 |
| **考生编排** | 管理员设置考试资料、上载 TXT 日程、系统验证派位 |
| **考试文件生成** | 自动生成日程表、准考证、成绩单 |
| **考后成绩处理** | 上载 OMR 成绩或人工录入，生成汇总数据 |
| **财务报告** | 生成各类收费及人数统计报告 |

图例说明：

| 标识 | 含义 |
|---|---|
| **系统步骤** | 系统自动执行 |
| **步骤分支** | 根据条件进行的分支判断 |
| **Must have** | 必须实现的功能 |
| **视乎情况** | 按实际情况需进行的步骤 |

---

## 2. 使用角色

| 角色 | 职责 |
|---|---|
| **申请人（保送教师/学校）** | 填写报名表、上载证明文件、缴费 |
| **管理员** | 设置考试资料、上载日程、处理成绩、生成报告 |
| **系统** | 自动验证数据、生成文件、发送通知 |

## 3. 流程一：在线报名流程

> 对应文件：`BDA - 報名流程_11Nov2025.xlsx`  
> 系统功能：Maintain Exam Information For Online Registration（COMM30005）、Maintain Grade（BDA025F）、Maintain Schedule（BDA20035）

---

### 3.1 申请人基本资料

**步骤 1：** 申请人进入报名页面，阅读报考简章及考试规则。

**步骤 2：** 申请人提供基本资料，以电邮地址开立账户，供日后查阅报名资料及下载文件。

> ⚠️ 如系统没有即时认证功能，系统须要求申请人输入电邮地址两次。

申请人须填写以下资料：

| 字段 | 说明 |
|---|---|
| 保送教师中文姓名 | 必填 |
| 保送教师英文姓名 | 必填 |
| 保送学校中文名称 | 如有 |
| 保送教师英文通讯地址 | 必填 |
| 保送教师联络电话 | 必填 |
| 保送教师电邮地址 | 必填，用于开立账户 |

---

### 3.2 选择考试类别及场地

**步骤 3：** 选择考试类别。

> ⚠️ 申请人**不可**同时选择两种考试类别。

目前开放的考试类别为**香港舞蹈总会**，细分如下：

#### 第 1–12 级考试

- 申请人**不可**同时选择两个考试场地类别
- 选择最多 **3 个考试日期**（第一选择、第二选择、第三选择）

#### 第 13 级 A/B 考试

申请人须选择场地类型：

**情况 A：自行安排场地**

1. 填写考场中文及英文地址
2. 选择该场地是否曾获香港舞蹈总会认可：
   - **是** → 上载考试场地平面图
   - **否** → 上载考试场地平面图
3. 选择最多 3 个考试日期

**情况 B：香港舞蹈总会场地**

- 选择最多 3 个考试日期

#### Joint Exam

- 标示是否与其他老师举行 Joint Exam（勾选 Check Box）
- 如是，填写其他保送教师姓名

---

### 3.3 填写考生资料

**步骤 4：** 申请人填写每一考试组别的考试级别、考生人数及考生个人资料。

- 申请人每次可**输入**或**利用模板导入**最多 **30 组**考生资料
- 管理员可自行设定每份考试报名表的**组别上限**
- 系统容许申请人替每个考生**自行填写组别及考生编号**（因考生考试时站位考量，例如：第 1 级考试即使只有 3 名考生，考生编号亦可选择 01、03、05）

每位考生须填写：

| 字段 | 说明 |
|---|---|
| 考生中文姓名 | 必填 |
| 考生英文姓名 | 必填 |
| 考生国籍 | 必填 |
| 考生性别 | 必填 |
| 身份证明文件类别 | 香港身份证 / 护照 / 出生证明书 / 其他 |
| 身份证明文件编号 | 必填 |

**上载考生身份证明文件副本：**

- 接受格式：JPEG / JPG / GIF / BMP / PDF / TIFF
- 文件大小：不大于 1MB
- 申请人可即时预览已上载的文件

---

### 3.4 年龄验证

**步骤 5：** 填写考生出生日期，系统自动计算年龄。

> 📌 考生岁数以考试**季度**计算：
> - 春季：以 **6 月 30 日**为准
> - 秋季：以 **12 月 31 日**为准

年龄验证逻辑如下：

#### 第 1–10 级考生

```
计算年龄
  ├── 符合年龄要求 → 可报考 ✅
  └── 不符合年龄要求 → 不可报考 ❌
      系统应提示申请人，考生不符合考试级别的年龄规定
```

#### 第 11–13 级考生

```
计算年龄
  ├── 符合年龄要求 → 可报考 ✅
  └── 不符合年龄要求 → 仍可报考（特殊情况）⚠️
```

#### 同一季度报考多科检查

系统检查考生是否于**同一考试季度**报考多于一个考试科目/级别：

- **是** → 系统应提示申请人
- **否** → 正常继续

> 管理员页面可另行生成报告，显示于同一季度报考多科的考生名单。

---

### 3.5 上载证明文件

**步骤 6：** 上载相关证明文件。

#### 教师证书及 HKDF 会员证

| 文件 | 格式要求 |
|---|---|
| 教师证书 | JPEG / JPG / GIF / BMP / PDF / TIFF，不大于 1MB |
| HKDF 会员证 | JPEG / JPG / GIF / BMP / PDF / TIFF，不大于 1MB |

- 申请人可即时预览已上载的文件

#### 报名资格证明（第 11–13 级）

| 报考级别 | 须提供 |
|---|---|
| 第 11 级 | 第 1–10 级考试证书 |
| 第 12 级 | 第 1–11 级考试证书 |
| 第 13 级 | 第 1–12 级考试证书 |

- 接受格式：JPEG / JPG / GIF / BMP / PDF / TIFF，不大于 1MB
- 申请人可即时预览已上载的文件

---

### 3.6 第 13 级专属：上载考试组别编排表

**步骤 7（仅第 13 级）：** 申请人上载考试组别编排表。

- 接受格式：图像文件 / PDF / ZIP

系统容许申请人替每个考生自行填写组别及考生编号，以配合考生考试时的站位安排。

- 申请人可即时预览已上载的文件

---

### 3.7 费用计算与确认

**步骤 8：** 系统列出管理人设定的报名及考试条款，要求申请人确认。

**步骤 9：** 系统判断是否于**逾期报名期间**递交：

- **是** → 系统收取逾期报名费：**港币 $511 / 每位考生**
- **否** → 按正常费用计算

**步骤 10：** 系统自动计算费用，规则如下：

1. **场地附加费**：系统收取每组的场地附加费用（按考试级别计算）
2. **人数不足附加费**：如每组考生人数不足一半，系统须额外收取附加费用（按考试级别计算）

> 详细费率请参考《BDA - 考试费用、附加费用及年龄限制》文件

**步骤 11：** 系统列出所有已输入的资料及每项收费明细（每个考试组别的考试收费及附加费用），要求申请人最终确认。

---

### 3.8 缴费及报名完成

**步骤 12：** 申请人缴交考试费及附加费用（如有）。

**步骤 13：** 系统发出**报名确认电邮**。

> - 电邮内容由管理人于系统中设定
> - 管理人可于系统直接查看**个别发送状态**
> - 管理人可于系统**重发**确认电邮

**步骤 14：** 报名完成 ✅

### 4.1 管理员设置阶段

| 场地类型 | 编号规则 | 示例 |
|---|---|---|
| 香港舞蹈总会 & 第 13 级 A/B 考试 | AA[考试年份] + B[考试月份] + CCC[报名顺序] | `243001` |
| 自行安排场地 | AA[考试年份] + B[考试月份] + CCC[报名顺序] | `244001` |

> ⚠️ 两种选项的考试发生在**不同月份**，故可由月份编号区分。

### 4.2 上载 TXT 考试日程

**步骤：** 管理员以 **TXT 格式**上载考试日程。

TXT 文件须按以下次序排列字段：

| 顺序 | 字段 |
|---|---|
| 1 | 考试日期 |
| 2 | 考试场地编号 |
| 3 | 考试场地类别 |
| 4 | 考官编号 |
| 5 | 申请人编号（由管理人编排） |
| 6 | 考试当天全日时间表的次序 |
| 7 | 考试组别 |
| 8 | 每一组考试开始及完结时间 |
| 9 | 考试级别 |
| 10 | 每一考试组别的考试长度 |

**相关功能：**
- Upload Template（BDA20065）

---

### 4.3 系统派位验证

上载 TXT 后，系统执行**两轮验证**：

#### 第一轮：重复检查

```
系统检查：每一考试日 × 每一考试场地 × 每组考生
         （组别及考生编号有否重复）
  ├── 有重复 → 系统提示管理员未能派位的考生名单
  │            → 管理员重新上载 TXT 档案
  └── 没有重复 → 进入第二轮验证
```

#### 第二轮：派位完整性检查

```
系统检查：所有考生是否已完成派位
  ├── 尚有考生未能派位 → 系统提示未能派位的考生名单
  │                     → 管理员重新上载 TXT 档案
  └── 所有考生成功派位 → 系统再次检查组别及考生编号有否重复
                        → 编排完成 ✅
                        → 编排结果可输出为 Excel 及考试日程
```
